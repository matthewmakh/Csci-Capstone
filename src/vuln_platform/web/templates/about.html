How this platform works

A four-agent pipeline that turns a target IP into a triaged, LLM-reasoned pentest report. Each stage hands off enriched data to the next, and every Claude call is recorded for review.

Ethics & authorization model

  • Every scan requires a scope file [?]: authorized CIDR ranges plus a signed attestation (authorized_by, date, statement).
  • Targets are cross-checked against the scope before any packet is sent. Out-of-scope targets fail-fast with a non-zero exit code.
  • The CFAA (18 U.S.C. § 1030) and equivalent foreign statutes treat unauthorized scanning as a federal crime. The scope-file requirement is the platform's primary safeguard.
  • Every Claude API call is recorded in audit.jsonl with prompt SHA-256s, response text, model, and token usage. The audit log viewer surfaces this directly.
